# 学科渲染调研：数学（K-12）

## 1）学科渲染特征概述

数学题视觉呈现高度依赖“符号精度 + 图形精度”。  
K-12 全学段从算术到微积分，核心痛点是：

- 同一题面中混合出现分式、根式、函数记号、几何图形、统计图。
- 图形与文字条件强耦合（点位、角标、长度标注一旦偏差就会误导解题）。
- 作业纸样式对行距、对齐、题号缩进敏感，普通网页排版易“看起来不像试卷”。
- 高中阶段出现大量组合表达（极限 + 导数 + 参数方程 + 图像性质），对渲染稳定性要求高。

## 2）前端渲染范围（纯文本公式类）

### 2.1 必须且只能走前端渲染的公式/符号类型（按学段）

小学（1-6 年级）：

- 四则运算表达式、括号嵌套、分数与带分数。
- 小数、百分数、比与比例、单位换算记号。
- 简单方程与未知数符号（如 `x`、`□`）。

初中（7-9 年级）：

- 整式/分式/根式运算表达式。
- 一元一次/一元二次方程与不等式组的文本表达。
- 函数记号（`y=kx+b`、`y=ax^2+bx+c`）及点坐标文本。
- 概率与统计的公式表达（平均数、方差基础记号）。

高中（10-12 年级）：

- 集合符号、逻辑符号、命题量词。
- 函数与反函数记号、导数与积分基础式、数列求和公式。
- 三角函数恒等式、向量代数表达、复数代数式。
- 解析几何方程文本（圆锥曲线方程、参数方程、极坐标方程）。

判定原则（数学）：

- 只要不需要“空间位置关系”即可前端渲染。
- 只要不需要“几何构型准确绘制”即可前端渲染。
- 只要是“单行或可分行的符号表达式”全部前端渲染。

### 2.2 标准 LaTeX 语法示例

```latex
% 分数与根式
\frac{3}{4} + \sqrt{2}

% 一元二次方程
ax^2 + bx + c = 0,\quad x=\frac{-b\pm\sqrt{b^2-4ac}}{2a}

% 三角恒等式
\sin^2\theta+\cos^2\theta=1

% 导数与积分
f'(x)=\lim_{h\to 0}\frac{f(x+h)-f(x)}{h},\quad
\int_a^b f(x)\,\mathrm{d}x

% 向量与复数
\vec{a}\cdot\vec{b}=|\vec{a}||\vec{b}|\cos\theta,\quad
z=a+bi
```

### 2.3 前端渲染性能评估与宏包建议

- 默认引擎：KaTeX（静态渲染速度快，适合题干密集展示）。
- 兼容兜底：MathJax（处理少量 KaTeX 不支持语法）。
- 数学通用宏包：KaTeX/MathJax 内建 TeX 数学能力已覆盖大部分 K-12 场景。
- 性能策略：
  - 题干首屏只渲染可视区公式，滚动区延迟渲染。
  - 公式 AST 缓存（同题重复出现时直接复用 HTML）。
  - 对超长表达式启用分段渲染与软换行策略。

## 3）后端渲染范围（复杂图形类）

### 3.1 K-12 可能出现的图形类型（全学段穷举）

- 小学：长度/面积示意图、条形图、折线图、扇形图、简单几何拼接图、平移旋转轴对称示意。
- 初中：三角形与四边形构型图、全等/相似辅助线图、圆与切线图、函数图像（一次/二次/反比例）、统计图升级版。
- 高中：立体几何（棱柱/棱锥/球）与截面图、空间向量示意、圆锥曲线图像、参数方程/极坐标图、导函数与原函数关系图、概率树与分布示意图。

### 3.2 图形类型 -> 后端技术栈建议

- 平面几何作图（角、垂线、中点、辅助线）：
  - `TikZ + tkz-euclide`
- 函数图像/坐标系/统计曲线：
  - `pgfplots`（LaTeX 原生）或 `Matplotlib`（Python）输出 SVG
- 立体几何与截面：
  - `TikZ-3dplot` / `Asymptote`（复杂空间关系）
- 概率树、流程框图：
  - `TikZ` 或 `Graphviz`（节点边关系）
- 作业纸内统一标注（A/B/C 点、角度弧、长度标签）：
  - `TikZ` 统一字形与线宽，避免多引擎风格冲突

### 3.3 后端渲染输出规范

- 输出格式统一为 SVG。
- 题干与图形通过 `figure_id` 绑定，避免前端错位注入。
- SVG 需保留可选文本层（便于无障碍与检索），同时锁定主要几何路径。

## 4）边界与特殊情况处理（Edge Cases）

- 边界 1：简单坐标点列图（仅少量点）  
  决断：若仅用于“文字说明”，可前端；若用于严谨解题（斜率、截距、对称），必须后端。

- 边界 2：分段函数“表达式 + 小示意图”  
  决断：表达式前端渲染；示意图后端渲染并绑定同题。

- 边界 3：表格化统计题  
  决断：纯数据表可前端；凡包含统计图（柱状/箱线/回归趋势）走后端。

- 边界 4：复杂连分式、矩阵、多行推导  
  决断：仍属纯符号公式，优先前端（MathJax 兜底），不进入图形后端。

- 边界 5：几何图中仅“一个三角形”  
  决断：即便图形简单，只要是解题核心构型，统一后端，确保边角标注和打印比例稳定。

## 附：数学学科落地建议

- 建立“数学图形 DSL”，至少包含点、线、圆、弧、角标、长度标、坐标轴 8 类原语。
- 每道生成题先过“图文一致性检查”：题干出现的点/线/角必须在图中可定位。
- 对常见模板（一次函数图、圆幂、相似三角形）做 SVG 模板缓存，降低渲染时延。

# 学科渲染调研：化学（K-12）

## 1）学科渲染特征概述

化学题面呈现的难点在于“上下标密集 + 反应条件 + 结构图”。  
典型痛点：

- 同一题面常同时出现化学式、离子方程式、实验装置图、有机结构式。
- 元素下标、离子电荷、状态符号、反应箭头与条件标注极易排版错位。
- 有机题和实验题往往“图文联动”，图错即题废。

## 2）前端渲染范围（纯文本公式类）

### 2.1 必须且只能走前端渲染的公式/符号类型（按学段）

初中：

- 元素符号与化学式（`H_2O`、`CO_2`）。
- 简单化学方程式与配平结果文本。
- 相对原子质量、质量守恒相关代数表达。

高中：

- 离子方程式、氧化还原电子转移文本表示。
- 化学平衡常数表达（`K_c`、`K_p`）、电离常数、溶度积。
- 物质的量、气体状态、浓度计算公式表达。
- 电化学、电极反应式文本表达。

必须前端的判定：

- 仅符号排列即可表达完整语义（不涉及空间拓扑）时，全部前端渲染。
- 包含大量上下标、反应条件，但无结构拓扑图，也仍是前端范围。

### 2.2 标准 LaTeX 语法示例

```latex
% 需要 mhchem 扩展
\ce{2H2 + O2 -> 2H2O}
\ce{Fe^{3+} + 3OH^- -> Fe(OH)3 v}
\ce{CH3COOH <=> CH3COO^- + H^+}

% 常见计算表达
n=\frac{m}{M},\quad c=\frac{n}{V}

% 平衡常数
K_c=\frac{[\ce{NH3}]^2}{[\ce{N2}][\ce{H2}]^3}
```

### 2.3 前端渲染性能评估与宏包建议

- 关键宏包：`mhchem`（化学式与反应式核心语法支持）。
- 引擎建议：
  - KaTeX + mhchem（高性能主路径）
  - MathJax + mhchem（复杂语法兜底）
- 性能策略：
  - 反应式按“句段”拆分渲染，降低单节点复杂度。
  - 对高频方程式（电离、沉淀、燃烧）建立缓存片段库。

## 3）后端渲染范围（复杂图形类）

### 3.1 K-12 图形类型穷举

- 初中实验图：
  - 气体制取与收集装置
  - 过滤、蒸发、结晶、分液等基本操作示意
- 高中实验图：
  - 滴定装置、原电池/电解池装置、尾气吸收装置
  - 实验流程图、误差分析示意图
- 结构化学图：
  - 有机物结构简式/键线式
  - 分子空间构型示意（课标范围内）
  - 晶体结构局部示意、配位结构示意
- 数据图：
  - 滴定曲线、溶解度曲线、反应速率曲线

### 3.2 图形 -> 后端技术栈映射

- 有机结构式、反应路径结构图：
  - `chemfig`（LaTeX）
  - `RDKit`（从 SMILES 生成 2D 结构图）
- 实验装置图：
  - `TikZ`（可控图元拼装）
- 数据曲线（滴定/溶解度/速率）：
  - `pgfplots` / `Matplotlib`
- 晶体与空间示意：
  - `TikZ`（简化示意）+ 必要时 `Python` 生成几何投影

### 3.3 输出要求

- 图像统一 SVG，保留化学标签可检索文本层。
- 有机结构图必须满足键角、连接关系可读，不可仅凭装饰效果。
- 图中反应条件（温度、催化剂）需与题干保持一致。

## 4）边界与特殊情况处理（Edge Cases）

- 边界 1：非常复杂的离子方程式阵列（多行、上下标密集）  
  决断：仍归前端公式（MathJax 兜底），不误入后端图形。

- 边界 2：化学方程式中包含“结构简式”  
  决断：纯文本简式可前端；若出现键线拓扑结构，必须后端。

- 边界 3：实验题只有“装置名称列表”无图  
  决断：若题意依赖器材连接顺序，应自动补后端装置图。

- 边界 4：数据表 + 曲线题  
  决断：表格可前端，曲线必须后端（坐标精度要求高）。

- 边界 5：化学平衡图像题（浓度-时间）  
  决断：图像一律后端，公式说明前端并行渲染。

## 附：化学学科落地建议

- 引入“化学语法校验器”：元素守恒、电荷守恒、状态符号完整性。
- 结构图采用 RDKit + chemfig 双路策略：优先 RDKit 自动布局，复杂教材样式走 chemfig 模板。
- 题图一致性规则需覆盖：物质种类、反应条件、装置功能、曲线含义。

# 学科渲染调研：历史（K-12）

## 1）学科渲染特征概述

历史题核心是“时间 + 空间 + 因果”的复合表达。  
排版痛点：

- 同题常并列出现时间轴、地图、史料摘录、统计图。
- 历史事件前后关系与并发关系对时间轴精度要求高。
- 战争与贸易题常依赖路线图，图层错位会直接误导判断。

## 2）前端渲染范围（纯文本公式类）

### 2.1 必须且只能走前端渲染的符号/文本表达（按学段）

初中：

- 朝代、世纪、年代的文本标记。
- 简单时间换算表达（公元前后、年份间隔）。
- 史实要点编号与关键词标注。

高中：

- 时间跨度计算表达（如“持续 n 年”）。
- 统计数据文本计算（占比、变化率）。
- 史学论证中的逻辑连接符号（因果、并列、转折）。

统一规则：

- 不涉及图形空间布局的时间/数据表达，一律前端渲染。
- 仅文字史料、问答题干、论证框架前端渲染。

### 2.2 标准 LaTeX 示例

```latex
\text{公元前}221\text{年}

\Delta t = t_2 - t_1

r=\frac{V_t-V_0}{V_0}
```

### 2.3 前端渲染性能评估与宏包建议

- 历史学科对数学引擎依赖弱，KaTeX 用于少量符号即可。
- 主要性能策略：
  - 长材料文本分块渲染。
  - 题干与史料引用采用可折叠结构，减少首屏 DOM 压力。

## 3）后端渲染范围（复杂图形类）

### 3.1 K-12 图形类型穷举

- 初中：
  - 基础时间轴（朝代更替、重大事件顺序）
  - 历史地图（疆域、迁徙、路线）
  - 表格化对比图（制度对比、事件对比）
- 高中：
  - 多线并行时间轴（中外历史并行）
  - 战争进程图、外交关系图、贸易网络图
  - 革命/改革过程流程图
  - 人物关系网络与思想传播路径图
  - 数据统计图（工业产值、人口变化等）

### 3.2 图形 -> 后端技术栈映射

- 时间轴与并行事件图：
  - `Mermaid timeline` / `Gantt`（后端转 SVG）
- 关系网络图：
  - `Graphviz`
- 历史地图与路线图：
  - `Cartopy` / `D3-geo`（后端静态输出）
- 统计图：
  - `Matplotlib` / `ECharts SSR`

### 3.3 输出要求

- 时间轴需清晰标注尺度，避免时间密度误导。
- 历史地图需标注图例与关键地名，保证教学可读性。
- 史实关系图不得“艺术化扭曲”因果方向。

## 4）边界与特殊情况处理（Edge Cases）

- 边界 1：仅问“事件先后排序”且无图  
  决断：前端文本列表可满足。

- 边界 2：题干给出简短时间线（3 个事件）  
  决断：若仅记忆题可前端；若考察并发关系，后端时间轴更稳妥。

- 边界 3：历史地图只有一个箭头路线  
  决断：只要涉及地理路径判断，统一后端地图。

- 边界 4：史料题附小统计图  
  决断：图形内容后端；史料文本前端。

- 边界 5：跨学科题（历史 + 经济数据）  
  决断：经济图表仍按后端统计图通道处理。

## 附：历史学科落地建议

- 先建设“时间轴模板库”：单线、双线并行、多线并行三种基础版式。
- 地图题建立地名词典和年代词典，做题图一致性检查。
- 对高中文综场景提供“材料-图形-小问”三向锚点映射，提高作答效率。

# 学科渲染调研：英语（K-12）

## 1）学科渲染特征概述

英语学科不是“数学公式型”，但在 K-12 题型中存在大量结构化版式需求：  
阅读理解、完形填空、语法填空、任务型阅读、写作与图文信息提取题等。  
常见痛点：

- 长文本 + 题目选项 + 表格/图示同页排版易错位。
- 音标、重音、语法树、信息结构图需要稳定字体与间距。
- 题干中的图表题（尤其跨学科英语题）需要图文精准对应。

## 2）前端渲染范围（纯文本公式类）

### 2.1 必须且只能走前端渲染的“符号/结构化文本”类型（按学段）

小学：

- 单词拼写、短语、基础句型、选择题选项排版。
- 音标字符与重音标识（纯文本层）。

初中：

- 时态、语态、从句标注（如括号、下划线、编号）。
- 完形填空空位编号、阅读题定位标记。

高中：

- 复杂句法标注（主谓宾、同位语、非谓语结构标记）。
- 段落逻辑符号（因果、转折、递进）文本标注。
- 图表类阅读中的数据表达式文本（百分比、时间序列文本）。

判定规则：

- 纯文字、音标、语法标注、编号体系全部前端渲染。
- 任何需要“关系网络可视化”或“图表空间布局”的内容转后端。

### 2.2 标准 LaTeX/标注示例

```latex
% 题号与空位标注
\text{(1)}\ \underline{\qquad}

% 语法结构提示
\text{Subject} + \text{Predicate} + \text{Object}

% 百分比与时间表达
70\% \text{ of students improved in 3 months}
```

### 2.3 前端渲染性能评估与宏包建议

- 英语题核心性能在长文本排版，不在数学公式引擎本身。
- 建议：
  - 前端使用稳定排版组件（段落、题号、选项块、空位块）。
  - 音标字符使用统一字体族并做 fallback。
  - 数学公式引擎只用于少量符号，不做主渲染路径。

## 3）后端渲染范围（复杂图形类）

### 3.1 K-12 图形类型穷举

- 小学/初中：
  - 看图说话配图（情景图）
  - 信息匹配图（路线图、简易流程图）
- 高中：
  - 阅读理解中的数据图（柱状图、折线图、饼图）
  - 语篇结构图（段落关系、论证结构）
  - 句法树/成分树（教学与训练场景）
  - 跨学科文本中的地图/流程图/实验图（英语载体题）

### 3.2 图形 -> 后端技术栈映射

- 语篇结构图、流程图：
  - `Mermaid` / `Graphviz` 后端转 SVG
- 句法树：
  - `TikZ-qtree` / `forest`（LaTeX 句法树宏包）
- 数据图表：
  - `Matplotlib` / `ECharts SSR`
- 情景插图（生成型）：
  - 图像生成服务（后端生产 SVG/PNG，按题干实体约束）

### 3.3 输出要求

- 英文行距、段间距、选项缩进需符合学校英语卷版式。
- 图表类题必须在图下提供最小文字说明，避免语义脱节。
- 句法树节点标签与题干语法术语一致（避免中英混乱）。

## 4）边界与特殊情况处理（Edge Cases）

- 边界 1：简单 2x2 词汇匹配表  
  决断：可前端表格渲染。

- 边界 2：长篇阅读中插入一个小图标  
  决断：装饰性图标可前端；只要参与解题判断，走后端可控图。

- 边界 3：英语题中包含数学/地理图表  
  决断：按“图表学科规则”后端渲染，不因学科标签为英语而放前端。

- 边界 4：语法树仅两层  
  决断：若用于语法判断训练，统一后端生成，保持模板一致。

- 边界 5：阅读理解中的表格非常复杂（跨页）  
  决断：后端排版并输出 SVG/PDF 片段，前端仅展示。

## 附：英语学科落地建议

- 先标准化题型组件：阅读、完形、语法填空、写作。
- 建立“英语图表跨学科路由器”，自动把图表类请求送后端。
- 对移动端做分段折叠与题目锚点导航，解决长文本可读性。

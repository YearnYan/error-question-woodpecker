# ARCHITECTURE 研究：错题啄木鸟

## 组件边界

- Web 客户端（React）
  - 左侧：上传、原题展示、分析结果
  - 右侧：作业预览、打印导出
  - 公式前端渲染层（KaTeX/MathJax）

- 应用网关（API）
  - 用户请求编排（上传、分析、生成、查询任务）
  - 统一鉴权、限流、日志

- 题目理解服务（AI + 规则）
  - OCR/结构化抽取
  - 学科识别、考点抽取、分析生成

- 题目生成服务（AI + 模板）
  - 三类题构造
  - 学科提示词与质量校验

- 图形渲染服务（后端）
  - 路由到 TikZ/pgfplots/circuitikz/chemfig/RDKit/Cartopy/Mermaid
  - 生成 SVG 并写入对象存储

- 作业排版服务
  - 作业纸模板（页眉、题号、分页、留白）
  - 图文组装、导出 PDF

- 任务与缓存层
  - 队列（渲染任务异步化）
  - 缓存（图形 hash 命中）

## 数据流

1. 用户上传错题 -> 网关保存原图并触发抽取。  
2. 抽取结果返回左侧展示 -> 用户点击“错题分析”。  
3. 分析完成后写回左侧 -> 用户点击“生成举一反三作业”。  
4. 题目生成服务产出三类题（含图形描述 DSL）。  
5. 图形渲染服务按学科 DSL 生成 SVG，写入 CDN。  
6. 排版服务组装作业纸并返回右侧预览。  
7. 用户可导出 PDF/打印。

## 推荐构建顺序

1. 上传与抽取链路  
2. 错题分析  
3. 三类题生成  
4. 图形渲染路由与 SVG 资产化  
5. 作业纸模板与导出  
6. 性能稳定性治理（缓存、队列、监控）
